# 📊 داشبورد تحلیل داده | Data Analytics Dashboard

یک داشبورد تحلیل و مصورسازی داده‌ی کاملاً **بدون نیاز به نصب** و **مبتنی بر مرورگر**، با پشتیبانی کامل از زبان فارسی و چیدمان راست‌به‌چپ (RTL). فایل‌های CSV و Excel خود را بارگذاری کنید و در چند ثانیه به شاخص‌های کلیدی، آمار توصیفی، تحلیل همبستگی، نمودارهای پویا، فیلتر، جست‌وجو و خروجی‌های حرفه‌ای دست پیدا کنید — چیزی شبیه یک نسخه‌ی کوچک از Power BI / Tableau.

> همه‌چیز در مرورگر شما اجرا می‌شود؛ هیچ داده‌ای به سروری ارسال نمی‌شود.

## 📖 Usage Guide | راهنمای استفاده

1. **بارگذاری داده:** فایل `CSV`/`Excel` را بکشید و رها کنید، یا روی ناحیه‌ی آپلود کلیک کنید. برای آزمایش سریع، روی **«استفاده از مجموعه‌داده نمونه»** بزنید.
2. **مرور شاخص‌ها:** بلافاصله کارت‌های KPI و پیش‌نمایش داده نمایش داده می‌شوند.
3. **کاوش با تب‌ها:** بین تب‌های *نمای کلی، مقادیر گمشده، همبستگی، نمودارساز، شمارش مقادیر، گروه‌بندی، کیفیت داده* جابه‌جا شوید (با کلیک یا کلیدهای جهت‌دار ◀ ▶).
4. **نوار ابزار جدول:** بالای جدول داده یک نوار ابزار حرفه‌ای قرار دارد — **«📊 نمودار جدید»** (پرش به نمودارساز)، **«🔍 جستجو»** و **«⚙️ فیلتر»** (پنل‌های بازشونده)، **«🔄 بازنشانی»** (پاک کردن جست‌وجو و همه فیلترها) و منوی **«⬇️ خروجی»**. زیر نوار ابزار، خط وضعیت زنده تعداد ردیف‌های نمایش‌داده‌شده را نشان می‌دهد.
5. **فیلتر و جست‌وجو:** پنل جست‌وجو/فیلتر را باز کنید و شرط‌ها را اعمال کنید — **همه‌ی جدول‌ها، شاخص‌ها و نمودارها به‌صورت زنده به‌روزرسانی می‌شوند.** نشانگرِ دکمه‌ها وضعیت فعال (جست‌وجوی فعال یا تعداد فیلترها) را نشان می‌دهد.
6. **ساخت نمودار:** در «نمودارساز» نوع نمودار، محورها و روش تجمیع را انتخاب کنید.
7. **خروجی گرفتن:** از منوی «خروجی» در نوار ابزار جدول، داده را به Excel/CSV، نمودارها را به PNG/SVG و گزارش کامل را به PDF بگیرید (در پنجره‌ی چاپ، مقصد را «ذخیره به‌صورت PDF» انتخاب کنید).
8. **حالت تاریک:** از دکمه‌ی بالا-چپ هدر استفاده کنید (انتخاب شما ذخیره می‌شود).

### معماری

- **بدون build / بدون باندلر:** همه‌ی ماژول‌ها به‌صورت `<script>` کلاسیک بارگذاری می‌شوند و یک فضای‌نام سراسری `window.App` را به‌اشتراک می‌گذارند. این انتخاب، اجرای مستقیم از `file://` و آفلاین را حفظ می‌کند.
- **حالت متمرکز:** `state.js` تنها منبع حقیقت است؛ داده‌ی خام، نمای فیلترشده (memoised)، و نوع ستون‌ها (کش‌شده) را نگه می‌دارد و با الگوی pub/sub بخش‌ها را هنگام تغییر فیلتر/جست‌وجو به‌روزرسانی می‌کند.

#### Architecture Diagram | نمودار معماری

```
            ┌──────────────┐      ┌──────────────┐
  فایل ───▶ │  upload.js   │      │   main.js    │ (bootstrap)
 CSV/XLSX   │ parse+validate│      └──────┬───────┘
            └──────┬───────┘             │ setup()
                   │ setData()           ▼
                   ▼              ┌──────────────┐
            ┌──────────────┐     │ dashboard.js │ ارکستراتور
            │   state.js   │◀────│ KPI/تب‌ها/تم  │
            │ راو + نمای    │     └──────┬───────┘
            │ memoised +کش  │            │ render() / subscribe()
            │  pub/sub      │            ▼
            └──────┬───────┘   ┌───────────────────────────────┐
   getView()/      │ refresh() │ بخش‌ها (هر کدام یک ماژول):       │
   colValues()     ├──────────▶│ statistics · correlation       │
                   │           │ charts · filters · insights    │
                   │           │ export · ui (کامپوننت‌ها)        │
                   │           └───────────────────────────────┘
                   ▼
        تغییر فیلتر/جست‌وجو ─▶ refresh() ─▶ به‌روزرسانی زنده‌ی
        KPI، کارت بینش/اطلاعات، پیش‌نمایش و تب‌های داده‌محور
```

کتابخانه‌های بیرونی (محلی): **Plotly** (نمودار)، **PapaParse** (CSV)، **SheetJS/xlsx** (اکسل)، **Tailwind** (استایل).

#### Dashboard Rendering Flow | جریان رندر داشبورد

۱. `upload.js` فایل را پارس و اعتبارسنجی می‌کند و با `state.setData()` داده را در `state.js` می‌نشاند.
۲. `dashboard.render()` چیدمان را به‌ترتیب می‌سازد: نوار هویت (`dashHeader`) ← کارت‌های KPI (`dashSummary`) ← بینش‌ها (`insights`) ← نوار ابزار جدول (`toolbar`) ← جدول پیش‌نمایش (`preview`) ← تب‌ها.
۳. تب‌ها با **رندر تنبل (lazy)** پر می‌شوند؛ فقط تب فعال بلافاصله رندر می‌شود و بقیه هنگام باز شدن (تب‌های سنگین با اسپینر محلی).
۴. `render()` یک‌بار در `state.subscribe()` مشترک می‌شود؛ با هر تغییر فیلتر/جست‌وجو، `state.refresh()` هدر، KPI، بینش، پیش‌نمایش و تب‌های داده‌محور (`overview / missing / correlation / quality`) را زنده به‌روزرسانی می‌کند — تب‌های دارای حالت داخلی (نمودارساز/شمارش/گروه‌بندی) دست‌نخورده می‌مانند.

## ♿ Accessibility | دسترس‌پذیری

- **پیمایش تب‌ها با صفحه‌کلید:** نوار تب‌ها از الگوی WAI-ARIA `tablist` پیروی می‌کند؛ کلیدهای ◀ / ▶ بین تب‌ها (متناسب با RTL) و `Home`/`End` به ابتدا/انتها جابه‌جا می‌شوند و تب مقصد فعال شده و focus می‌گیرد.
- **سرستون‌های مرتب‌شونده با صفحه‌کلید:** هر سرستونِ قابل‌مرتب‌سازی focusable است (`tabindex=0`، `role="columnheader"`) و با `Enter` یا `Space` مرتب می‌شود؛ ترتیب فعلی با `aria-sort` به فناوری‌های کمکی اعلام می‌شود.
- **منوی خروجی:** الگوی `role="menu"` با focus چرخشی (▲ ▼ Home/End)، بستن با `Escape` و بازگشت focus به دکمه، و صفات `aria-haspopup`/`aria-expanded`.
- **پنل‌های جست‌وجو/فیلتر و ناحیه‌ی آپلود:** با صفحه‌کلید قابل‌فعال‌سازی (Enter/Space) و دارای حلقه‌ی focus قابل‌رؤیت؛ `Escape` پنل‌ها را می‌بندد.
- **RTL و `aria-live`:** چیدمان کامل راست‌به‌چپ؛ خط وضعیت جدول با `aria-live="polite"` تغییرات نما را اعلام می‌کند.

## ⚡ Performance | کارایی

- **صفحه‌بندی جدول:** هر صفحه فقط ۵۰ ردیف را به DOM اضافه می‌کند؛ مناسب برای مجموعه‌داده‌های ۵۰٬۰۰۰+ ردیفی.
- **کش و memoization:** نوع ستون‌ها یک‌بار هنگام بارگذاری محاسبه می‌شود؛ نمای فیلترشده تا تغییر بعدی کش می‌شود.
- **به‌روزرسانی بهینه‌ی نمودار:** استفاده از `Plotly.react`/`relayout` به‌جای بازسازی کامل؛ تغییر تم بدون بازسازی DOM، فقط رنگ نمودارها را به‌روز می‌کند.
- **debounce:** ورودی جست‌وجو برای جلوگیری از رندرهای اضافی debounce شده است.

## ⚠️ Known Limitations | محدودیت‌های شناخته‌شده

- **خروجی PDF** از طریق پنجره‌ی چاپ مرورگر تولید می‌شود (مقصد: «ذخیره به‌صورت PDF»)؛ نیازمند اجازه‌ی باز شدن پاپ‌آپ است.
- **جدول‌ها صفحه‌بندی‌شده‌اند** (۵۰ ردیف در هر صفحه) و از مجازی‌سازی کامل استفاده نمی‌کنند؛ برای ۵۰هزار ردیف روان است اما برای صدها هزار ردیف ممکن است کند شود.
- **تشخیص نوع ستون** مبتنی بر نمونه‌گیری از ۵۰ مقدار ابتدایی است؛ ستون‌های ناهمگن ممکن است گاهی نادرست دسته‌بندی شوند.
- **همبستگی** تنها از نوع پیرسون (خطی) است؛ روابط غیرخطی را اندازه نمی‌گیرد.
- **حافظه‌ی تقریبی** یک برآورد است (نمونه‌گیری‌شده)، نه اندازه‌گیری دقیق.
- پردازش به‌صورت تک‌رشته‌ای (بدون Web Worker) انجام می‌شود؛ فایل‌های بسیار بزرگ ممکن است هنگام پارس، رابط را لحظه‌ای مسدود کنند.
