# 📊 داشبورد تحلیل داده | Data Analytics Dashboard

یک داشبورد تحلیل و مصورسازی داده‌ی **مبتنی بر مرورگر** و **بدون نیاز به نصب**، با رابط کاربری کاملاً فارسی و راست‌به‌چپ (RTL). فایل CSV یا Excel خود را بارگذاری کنید تا در چند ثانیه به شاخص‌های کلیدی، آمار توصیفی، تحلیل همبستگی، نمودارهای پویا و خروجی‌های حرفه‌ای برسید — چیزی شبیه نسخه‌ای سبک از Power BI / Tableau.

> تمام پردازش در مرورگر شما انجام می‌شود؛ هیچ داده‌ای به سرور ارسال نمی‌گردد.

</div>

---

## ✨ امکانات | Features

<div dir="rtl">

- **بارگذاری داده** — پشتیبانی از CSV و Excel با کشیدن‌ورهاکردن، به‌همراه مجموعه‌داده‌ی نمونه
- **داشبورد تعاملی** — شاخص‌های کلیدی (KPI)، پیش‌نمایش داده و بینش‌های خودکار
- **آمار توصیفی** — count، mean، std، quartiles و تشخیص نوع داده
- **تحلیل همبستگی** — ماتریس و نقشه‌ی حرارتی پیرسون
- **نمودارساز** — میله‌ای، خطی، پراکندگی، دایره‌ای، هیستوگرام و جعبه‌ای با تجمیع
- **فیلتر و جست‌وجو** — فیلتر عددی/دسته‌ای و جست‌وجوی زنده‌ی سراسری
- **کیفیت داده** — امتیاز کیفیت، مقادیر گمشده، تکراری‌ها و شناسایی داده‌ی پرت
- **خروجی** — Excel، CSV، تصویر نمودار (PNG/SVG) و گزارش PDF
- **رابط واکنش‌گرا** — بهینه برای موبایل و دسکتاپ
- **حالت تاریک/روشن** — با ذخیره‌ی انتخاب کاربر
- **رابط فارسی RTL** — چیدمان کامل راست‌به‌چپ با فونت وزیرمتن
- **دسترس‌پذیری** — پیمایش با صفحه‌کلید و صفات ARIA

</div>

## 🛠️ فناوری‌ها | Technologies

- **Vanilla JavaScript (ES6+)** — بدون فریم‌ورک و بدون باندلر
- **HTML5** و **CSS3** — به‌همراه سیستم design token
- **Tailwind CSS** — نسخه‌ی محلی (Browser Runtime)
- **[Plotly.js](https://plotly.com/javascript/)** — نمودارها و نقشه‌ی حرارتی
- **[PapaParse](https://www.papaparse.com/)** — پارس CSV
- **[SheetJS](https://sheetjs.com/)** — خواندن و نوشتن Excel

<div dir="rtl">

تمام کتابخانه‌ها به‌صورت محلی در `libs/` قرار دارند و پروژه کاملاً آفلاین کار می‌کند.

</div>

---

## 🏗️ معماری | Architecture

<div dir="rtl">

پروژه به‌صورت ماژولار و بدون مرحله‌ی build سازمان‌دهی شده است. هر ماژول یک شیء را زیر فضای‌نام سراسری `window.App` ثبت می‌کند و به‌ترتیب وابستگی در `index.html` بارگذاری می‌شود.

</div>

| لایه | مسئولیت |
|------|---------|
| **Application Core** | راه‌اندازی، مدیریت حالت مرکزی و الگوی pub/sub |
| **Dashboard UI** | چیدمان، تب‌ها، نوار ابزار و اجزای ارائه‌ای |
| **Data Processing** | آمار توصیفی، همبستگی، کیفیت داده و بینش‌ها |
| **Visualization** | نمودارساز و رندر نمودارها با Plotly |
| **Export** | تولید خروجی Excel، CSV، تصویر و PDF |
| **Utilities** | کمک‌توابع DOM، قالب‌بندی و آماره‌های پایه |

<div dir="rtl">

**جریان کلی:** فایل بارگذاری و اعتبارسنجی می‌شود ← داده در حالت مرکزی می‌نشیند ← داشبورد رندر می‌شود ← هر تغییر فیلتر یا جست‌وجو، شاخص‌ها، نمودارها و جدول‌ها را به‌صورت زنده به‌روزرسانی می‌کند.

</div>

## 🚀 راه‌اندازی | Getting Started

<div dir="rtl">

هیچ ابزار build یا وابستگی‌ای لازم نیست.

</div>

```bash
# دریافت پروژه
git clone <repo-url>
cd Data-Analytics-Dashboard

# اجرا با یک سرور محلی
python -m http.server 8000
```

<div dir="rtl">

سپس به `http://localhost:8000` بروید و یک فایل CSV یا Excel بارگذاری کنید (یا از مجموعه‌داده‌ی نمونه استفاده کنید). اجرا با سرور محلی برای جلوگیری از محدودیت‌های `file://` توصیه می‌شود.

تست اختیاری منطق اصلی:

</div>

```bash
node test/smoke-test.js
```

## 🎯 فلسفه‌ی طراحی | Design Philosophy

<div dir="rtl">

- **Zero-build** — بدون باندلر یا مرحله‌ی کامپایل؛ اجرای مستقیم در مرورگر
- **Client-side only** — تمام پردازش محلی، بدون بک‌اند
- **Modular** — ماژول‌های تک‌مسئولیتی با فضای‌نام مشترک
- **Maintainable** — کد ساده و قابل‌نگهداری
- **Responsive-first** — طراحی برای موبایل و دسکتاپ
- **Accessibility-aware** — پیمایش با صفحه‌کلید و ARIA
- **Design-token based** — استایل‌دهی یکپارچه با توکن‌های طراحی

</div>
